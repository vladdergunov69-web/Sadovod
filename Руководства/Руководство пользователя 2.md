## 1. Исправление: оформление заказа

В предыдущей версии при нажатии кнопки «Подтвердить заказ» заказ не создавался — появлялось сообщение об ошибке. Теперь оформление работает корректно.

Как оформить заказ:
1. Добавьте товары в корзину и перейдите в неё.
2. Нажмите «Оформить заказ» (доступно после входа в систему).
3. Проверьте данные получателя (ФИО и телефон подставляются из профиля), при необходимости укажите комментарий.
4. Нажмите «Подтвердить заказ».

После подтверждения заказу присваивается номер и статус «Новый», остатки товаров на складе уменьшаются, корзина очищается, а сам заказ появляется в разделе «Мои заказы».

## 2. Расширенные фильтры и сортировка в каталоге

На странице «Каталог» к привычным поиску по названию и фильтру по категории добавлены новые инструменты подбора товаров. Все они расположены в одной панели над списком товаров и работают совместно.

### Фильтр по цене

Поля «Цена от» и «Цена до» позволяют ограничить выдачу по стоимости. Можно заполнить оба поля (диапазон), либо только одно — например, указать лишь «Цена до», чтобы увидеть товары не дороже заданной суммы.

### Фильтр «Только в наличии»

Установите флажок «Только в наличии», чтобы в каталоге отображались исключительно товары, доступные на складе. Закончившиеся позиции будут скрыты.

### Сортировка

Выпадающий список «Сортировка» задаёт порядок отображения товаров:
- **По названию** — по алфавиту (выбрано по умолчанию);
- **Цена ↑** — от самых дешёвых к дорогим;
- **Цена ↓** — от самых дорогих к дешёвым;
- **По наличию** — сначала товары с наибольшим остатком.

После выбора параметров нажмите «Найти». Заданные фильтры и сортировка сохраняются при переходе по страницам каталога. Кнопка «Сброс» очищает все условия и возвращает каталог к исходному виду.

## 3. Административная панель: вкладка «Заказы»

Раздел доступен пользователям с ролью «Администратор». В меню административной панели появился пункт **«Заказы»**.

### Просмотр заказов

На странице отображается список всех заказов магазина: номер, дата и время оформления, покупатель (ФИО и email), состав заказа, итоговая сумма и текущий статус. Заказы отсортированы по дате — новые сверху, список разбит на страницы.

Над списком расположен фильтр по статусу: выберите нужный статус (Новый, Обработан, Выполнен, Отменён) или оставьте «Все» и нажмите «Найти».

[Рисунок 3 — Список заказов в административной панели с фильтром по статусу]

### Смена статуса заказа

Напротив каждого заказа находится выпадающий список со статусами. Выберите новый статус и нажмите кнопку «OK» — статус заказа обновится, а покупатель увидит изменение в своём разделе «Мои заказы».

Если покупатель ранее подписался на push-уведомления, при смене статуса ему автоматически придёт уведомление с новым статусом заказа. Если подписки нет, статус всё равно изменится — просто без уведомления.

## 4. Административная панель: вкладка «Пользователи»

Раздел доступен пользователям с ролью «Администратор». В меню административной панели появился пункт **«Пользователи»**.

### Просмотр пользователей

Отображается список всех учётных записей: идентификатор, логин (email), ФИО, телефон и текущая роль. Список разбит на страницы. Ваша собственная учётная запись отмечена меткой «вы».

[Рисунок 5 — Список пользователей с ролями]

### Смена роли

Напротив каждого пользователя расположен выпадающий список ролей («Администратор» или «Пользователь»). Выберите нужную роль и нажмите «OK» — роль обновится немедленно.

Назначение роли «Администратор» открывает пользователю доступ ко всей административной панели; возврат роли «Пользователь» этот доступ закрывает.

**Важно:** изменить роль собственной учётной записи нельзя — это защита от случайной потери доступа администратором. Для своей строки элементы смены роли скрыты, отображается пометка «недоступно для себя».

Все изменения ролей фиксируются в разделе «Аудит» (журнал изменений) с указанием старого и нового значения.
