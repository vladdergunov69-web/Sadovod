# Руководство пользователя (дополнение № 3)
## Интернет-магазин «Садовод»

Документ описывает **только новые возможности**, добавленные в приложение дополнением № 3: автоматическое развёртывание через Docker, модуль «Избранное» и раздел «Аудит входов». Базовые разделы сайта описаны в основном документе [«Руководство пользователя.md»](Руководство%20пользователя.md), а доработки предыдущего этапа — в [«Руководство пользователя 2.md»](Руководство%20пользователя%202.md).

Что нового:

- развёртывание магазина и базы данных через Docker одной командой (раздел 1, для администратора);
- модуль «Избранное» — отметка понравившихся товаров (раздел 2, для покупателя);
- раздел административной панели «Аудит входов» — контроль попыток входа в систему (раздел 3, для администратора).

---

## 1. Развёртывание через Docker (для администратора)

Этот раздел адресован администратору или владельцу магазина, который устанавливает приложение на новом компьютере или сервере, и не требует навыков программирования — только умение выполнять команды в командной строке.

### 1.1. Что такое Docker и зачем он нужен

Docker — программа, которая упаковывает приложение «Садовод» и базу данных в готовые изолированные «контейнеры» со всем необходимым внутри. Без Docker для установки магазина нужно было бы вручную устанавливать СУБД, настраивать её, устанавливать платформу .NET и переносить файлы приложения. С Docker всё это делается одной командой: Docker сам скачивает нужные компоненты, настраивает их совместную работу и запускает магазин.

### 1.2. Что нужно установить

Перед первым запуском на компьютер администратора необходимо установить **Docker Desktop** (для Windows или macOS) — бесплатную программу, которую можно скачать с официального сайта Docker. После установки и перезапуска компьютера Docker Desktop должен быть запущен (в Windows — значок кита в системном трее) перед выполнением любых команд из этого раздела.

[Рисунок 1 — Запущенный Docker Desktop]

### 1.3. Подготовка файла .env

В папке проекта находится файл-образец `.env.example` со списком настроек, которые нужно задать перед первым запуском. Чтобы подготовить рабочую конфигурацию:

1. Скопируйте файл `.env.example` и переименуйте копию в `.env` (без расширения `.example`).
2. Откройте `.env` в любом текстовом редакторе и заполните поля:
   - **пароль администратора базы данных** — придумайте собственный надёжный пароль (используется только внутри контейнеров, наружу не публикуется);
   - **строка подключения к базе данных** — как правило, можно оставить значение из образца без изменений, заменив в нём только пароль на тот, что указан в пункте выше;
   - **ключи для push-уведомлений (VAPID)** — если push-уведомления не используются, можно оставить эти поля пустыми;
   - **порт сайта** — порт, на котором будет доступен магазин в браузере (по умолчанию `8080`); изменяйте его только если этот порт уже занят другой программой на вашем компьютере.
3. Сохраните файл. Файл `.env` содержит реальные пароли — не передавайте его посторонним и не публикуйте в общедоступных местах.

[Рисунок 2 — Заполненный файл .env в текстовом редакторе]

### 1.4. Запуск магазина

Откройте командную строку (PowerShell или терминал) в папке проекта и выполните команду:

```
docker compose -p sadovod up -d --build
```

При первом запуске Docker скачивает необходимые компоненты из интернета и собирает приложение — это может занять несколько минут. После завершения команды на экране будет видно, что контейнеры созданы и запущены.

[Рисунок 3 — Вывод команды запуска в командной строке]

### 1.5. Проверка, что всё поднялось

Чтобы убедиться, что все три составляющие магазина (база данных, инициализация базы данных и сам сайт) запущены корректно, выполните:

```
docker compose -p sadovod ps
```

В списке должны отображаться контейнеры базы данных и сайта в состоянии «running» (работает); контейнер, который один раз настраивает базу данных при первом запуске, отображается как успешно завершённый.

[Рисунок 4 — Список запущенных контейнеров]

Затем откройте в браузере адрес `http://localhost:8080` (или с тем портом, который вы указали в `.env`) — должна открыться главная страница магазина «Садовод».

[Рисунок 5 — Главная страница магазина, открытая после запуска через Docker]

### 1.6. Где взять пароль администратора

При самом первом запуске (на пустой базе данных) приложение автоматически создаёт учётную запись администратора со случайно сгенерированным паролем и выводит логин и пароль в журнал работы сайта. Чтобы посмотреть этот журнал, выполните команду:

```
docker compose -p sadovod logs web
```

Найдите в выводе строки с логином и паролем администратора и сохраните их — повторно эта информация не выводится. При следующих запусках (если база данных уже создана) новая учётная запись не генерируется и сообщение не выводится повторно.

[Рисунок 6 — Журнал контейнера с сообщением о созданной учётной записи администратора]

### 1.7. Остановка и повторный запуск

Чтобы временно остановить магазин без потери данных, выполните:

```
docker compose -p sadovod down
```

Все товары, заказы, пользователи и избранное сохраняются. Чтобы запустить магазин снова, выполните ту же команду, что и при первом запуске:

```
docker compose -p sadovod up -d --build
```

Если же нужно полностью удалить магазин вместе со всеми данными (например, чтобы начать с чистой базы данных), используйте команду с дополнительным ключом — **это безвозвратно удалит все товары, заказы и учётные записи**:

```
docker compose -p sadovod down -v
```

## 2. Модуль «Избранное» (для покупателя)

«Избранное» позволяет отмечать понравившиеся товары, чтобы быстро вернуться к ним позже, не используя для этого корзину.

### 2.1. Как добавить товар в избранное

На каждой карточке товара в каталоге, а также на странице отдельного товара, расположен значок в виде сердечка «В избранное». Нажмите на него — товар будет добавлен в «Избранное», а значок изменит вид, показывая, что товар отмечен.

[Рисунок 7 — Карточка товара в каталоге со значком «В избранное»]

[Рисунок 8 — Значок «В избранное» на странице отдельного товара]

### 2.2. Как посмотреть список избранного

Раздел «Избранное» доступен из выпадающего меню учётной записи в шапке сайта — рядом с пунктом отображается количество отмеченных товаров. Откройте этот раздел, чтобы увидеть все отмеченные товары в виде привычных карточек каталога.

[Рисунок 9 — Выпадающее меню учётной записи с пунктом «Избранное» и счётчиком]

[Рисунок 10 — Страница «Избранное» со списком отмеченных товаров]

### 2.3. Как убрать товар из избранного

Чтобы убрать товар, нажмите на тот же значок сердечка ещё раз — на карточке товара в каталоге, на странице товара или прямо на странице «Избранное». Товар сразу исчезнет из списка избранного, а счётчик в меню уменьшится.

### 2.4. Как добавить избранный товар в корзину

На странице «Избранное» рядом с каждым товаром есть кнопка добавления в корзину — нажмите её, чтобы положить товар в корзину без перехода на страницу товара.

### 2.5. Если вы не вошли в систему

Если попытаться нажать на значок «В избранное», не выполнив вход в систему, вместо отметки товара откроется страница входа. После входа в систему вы сможете вернуться к работе с избранным.

### 2.6. Если товар удалён администратором

Если товар, который вы ранее отметили как избранный, был удалён администратором из каталога, он автоматически и без дополнительных действий исчезнет из вашего списка «Избранное».

## 3. Аудит входов (для администратора)

### 3.1. Зачем нужен этот раздел

Раздел «Аудит входов» помогает следить за безопасностью учётных записей магазина: он фиксирует каждую попытку входа в систему — успешную и неуспешную — и позволяет заметить, если кто-то пытается подобрать пароль к чужой учётной записи.

### 3.2. Где находится раздел

Раздел доступен в административной панели — новый пункт меню **«Аудит входов»**, рядом с уже существующим журналом аудита изменений товаров и пользователей.

[Рисунок 11 — Пункт меню «Аудит входов» в административной панели]

### 3.3. Что показывает таблица

В таблице отображаются записи о попытках входа: логин, которым пытались войти, результат попытки (успешно или неуспешно), IP-адрес, с которого выполнялась попытка, время входа и, если пользователь впоследствии вышел из системы, время выхода.

[Рисунок 12 — Таблица записей аудита входов]

### 3.4. Как пользоваться фильтрами

Над таблицей расположены поля для отбора записей: по логину (введите часть логина и нажмите «Найти»), по результату (только успешные или только неуспешные попытки), а также по диапазону дат (от и до). Список разбит на страницы — по 25 записей.

[Рисунок 13 — Панель фильтров раздела «Аудит входов»]

### 3.5. Что означает подсветка строк красным цветом

Если по одному и тому же логину подряд зафиксировано пять и более неуспешных попыток входа за последние 15 минут, соответствующие записи в таблице выделяются красным цветом — это признак возможного подбора пароля к учётной записи.

[Рисунок 14 — Строки журнала, выделенные красным как подозрительная активность]

Если вы заметили такую подсветку, рекомендуется проверить, не пытается ли кто-то посторонний подобрать пароль к отмеченной учётной записи, и при необходимости связаться с владельцем этой учётной записи или сменить ей пароль.
