автоматизация развёртывания приложения и базы данных средствами Docker (многоэтапный Dockerfile, docker-compose с отдельным сервисом инициализации схемы базы данных, healthcheck, именованные тома для сохранности данных); реализация модуля «Избранное» по заявке покупателя — новая таблица базы данных, сервисный слой, AJAX-взаимодействие без перезагрузки страницы, выпадающее меню учётной записи со счётчиком; реализация системы аудита входа пользователей — новая таблица журнала, переработка сервиса аутентификации с фиксацией попыток входа и времени выхода без журналирования паролей, раздел административной панели с фильтрами и обнаружением подозрительной активности; диагностика и устранение особенностей развёртывания Docker на операционной системе Windows при пути проекта с кириллическими символами; сквозное живое тестирование развёрнутого стенда и проверка отсутствия регрессий в существующем функционале; оформление технической и пользовательской документации по дополнению.

При прохождении этого этапа учебной практики я понял, что контейнеризация уже работающего приложения — это не только описание сервисов в одном файле конфигурации, но и внимательная проверка того, как окружение (операционная система, файловая система, кодировки путей) влияет на поведение инструментов сборки и монтирования. Здесь особенно важно не просто добиться запуска по инструкции, а убедиться, что результат воспроизводим на чистой среде.

Практика помогла мне закрепить навыки добавления нового функционала (модуль «Избранное», аудит входа) в действующее приложение с минимальными и заранее объявленными изменениями структуры базы данных, а также навыки практической диагностики через прямые запросы к контейнеризованной базе данных вместо опоры только на внешний вид страницы в браузере.
